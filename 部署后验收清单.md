# 796Helper 影视搜索部署后验收清单

> 目标：确认 `v2.0.2` 已真正生效，影视搜索优先打开真实网盘直链，旧缓存不再污染结果。

## 0. 验收前准备
- [ ] 使用无痕窗口，或先清空当前站点的 `sessionStorage`
- [ ] 确认前端已部署到最新页面资源
- [ ] 确认 Worker 已部署到最新代码
- [ ] 准备 2~3 个测试关键词：
  - [ ] `哪吒2`
  - [ ] `流浪地球2`
  - [ ] `三体`

## 1. Worker 健康检查
### 1.1 检查 `/health`
- [ ] 打开：`https://796helper-movie-search.clown8379.workers.dev/health`
- [ ] 期望返回 JSON
- [ ] `status` 为 `ok`
- [ ] `version` 为 `2.0.2`
- [ ] `sources` 包含 `pansearch` 和 `upyunso`

失败判定：
- 返回非 200
- `version` 不是 `2.0.2`
- 返回内容不是 JSON

## 2. API 结果验收
### 2.1 直链优先
- [ ] 打开：`https://796helper-movie-search.clown8379.workers.dev/api/search?keyword=哪吒2`
- [ ] 检查 `success` 为 `true`
- [ ] 检查 `data` 数组至少前 3 条里，优先出现真实网盘域名，而不是 `upyunso.com/search`
- [ ] 允许出现的直链域名示例：
  - [ ] `pan.baidu.com`
  - [ ] `drive.quark.cn`
  - [ ] `alipan.com` / `aliyundrive.com`
  - [ ] `pan.xunlei.com`
  - [ ] `cloud.189.cn`
  - [ ] `115.com`
  - [ ] `lanzou*`

### 2.2 字段完整性
- [ ] 每条结果至少包含：`title`、`source`、`sourceLabel`、`link`
- [ ] 若链接带 `pwd`，则 `code` 应同步返回
- [ ] `total` 大于 `0`

### 2.3 兜底行为
- [ ] 当直链不足时，结果中允许出现 `upyunso.com/search?...`
- [ ] 但 `UP云搜` 搜索页只能作为补充，不能覆盖已拿到的真实网盘直链

失败判定：
- 所有结果都变成 `upyunso.com/search`
- 明明存在直链资源，但前排结果仍全部是搜索页
- `success=false` 或 `data=[]`

## 3. 前端页面验收
### 3.1 首次打开页面
- [ ] 打开 796Helper 前端页面并进入 `影视搜索`
- [ ] 搜索 `哪吒2`
- [ ] 页面能正常展示结果卡片
- [ ] 结果卡片按钮文案与链接类型一致

### 3.2 打开链接行为
- [ ] 点击真实网盘结果的“打开网盘链接”按钮
- [ ] 新标签页应直接打开网盘站点，而不是跳到 `UP云搜` 搜索页
- [ ] 至少验证 2 条不同来源的结果

### 3.3 复制行为
- [ ] 对真实网盘结果点击“复制网盘链接”
- [ ] 剪贴板内容应为：直链 + 提取码（若存在）
- [ ] 对 `UP云搜` 兜底结果点击复制
- [ ] 剪贴板内容应为：资源标题 + 搜索页链接

### 3.4 来源筛选
- [ ] 切换 `全部 / 百度 / 夸克 / 阿里 / 迅雷` 等来源筛选
- [ ] 每次切换后结果能重新请求并正常展示
- [ ] 筛选后结果来源与所选类型基本一致

## 4. 缓存隔离验收
### 4.1 旧缓存清理
- [ ] 打开浏览器开发者工具
- [ ] 查看 `sessionStorage`
- [ ] 确认存在 `796h-mc-schema-version = 2.0.2`
- [ ] 确认新的缓存 key 前缀为 `796h-mc-2.0.2-`
- [ ] 确认旧前缀缓存未继续保留或未再被读取

### 4.2 重复搜索
- [ ] 同一关键词连续搜索两次
- [ ] 第二次结果仍应保持直链优先
- [ ] 刷新页面后再次搜索，不应回退成旧 `UP云搜` 搜索页结果

失败判定：
- `sessionStorage` 中仍主要命中旧 `796h-mc-` 结果
- 刷新后结果重新退化为旧搜索页链接

## 5. 异常与回归验收
### 5.1 空结果 / 超时
- [ ] 输入一个极冷门关键词
- [ ] 页面不应崩溃
- [ ] 若接口超时，应显示可理解的失败提示

### 5.2 兼容回归
- [ ] 普通搜索功能可用
- [ ] 路由切换后再次进入 `影视搜索`，按钮点击仍有效
- [ ] 其他页面功能未受影响

## 6. 通过标准
满足以下条件即可判定部署验收通过：
- [ ] `/health` 返回 `version = 2.0.2`
- [ ] API 前排结果已出现真实网盘直链
- [ ] 前端点击直链时，实际打开的是网盘站点
- [ ] `sessionStorage` 已切换到 `796h-mc-2.0.2-` 命名空间
- [ ] 无明显回归问题

## 7. 若验收失败，优先排查
1. 前端是否仍加载旧静态资源
2. Worker 是否未部署到最新版本
3. 浏览器是否仍保留旧 `sessionStorage`
4. PanSearch 上游页面结构是否再次变更
5. 当前测试关键词是否刚好无可用直链
