## 2. 默认站外搜索验收（无增强服务）

### 2.1 运行时配置确认
- [ ] 打开 `index.html`
- [ ] 确认 `window.__796HELPER_CONFIG__.movieSearch.apiBases` 为 `[]`
- [ ] 确认 `enableExternalFallback: true`

### 2.2 页面行为
- [ ] 打开前端页面并进入 `影视搜索`
- [ ] 欢迎态明确说明：默认会打开 `PanSearch` 搜索结果页
- [ ] 搜索 `哪吒2`
- [ ] 页面无需长时间等待即可展示结果卡片
- [ ] 结果顶部出现“当前未配置增强搜索服务”的说明
- [ ] 结果头文案为“已准备 X 个可继续打开的搜索入口”
- [ ] 默认只展示 `PanSearch` 搜索结果页入口

### 2.3 打开与复制
- [ ] 点击结果卡片的打开按钮
- [ ] 新标签页应进入 `PanSearch` 搜索结果页
- [ ] 点击复制按钮
- [ ] 剪贴板内容应为“资源标题 + 搜索页链接”

## 3. 增强搜索验收（可选）

> 说明：本阶段不再要求自定义域名。只要是兼容 `/api/search` 与 `/health` 的可访问服务地址，都可作为增强搜索服务。

### 3.1 配置方式
任选一种：
- [ ] 在 `index.html` 的 `apiBases` 数组中填入服务地址
- [ ] 或在页面“增强搜索设置”中手动填入服务地址并保存

### 3.2 健康检查
- [ ] 打开：`{增强服务地址}/health`
- [ ] 确认返回 HTTP 200
- [ ] 确认返回合法 JSON
- [ ] 确认 `status = "ok"`
- [ ] 若使用当前项目内置 Worker 实现，确认 `routes` 包含 `/api/search` 与 `/health`

### 3.3 真实直链结果
- [ ] 搜索 `哪吒2`
- [ ] 页面优先展示增强搜索结果，而不是直接回到搜索页入口
- [ ] 若结果存在直链，打开按钮应直达网盘站点
- [ ] 复制按钮应复制“直链 + 提取码（若存在）”
- [ ] 至少验证 2 条不同来源结果（如百度 / 夸克 / 阿里 / 迅雷）

### 3.4 增强失败自动回退
- [ ] 将增强服务地址临时改为无效地址，或让服务不可达
- [ ] 再次搜索关键词
- [ ] 页面应自动切换为 `PanSearch` 搜索页入口
- [ ] 顶部提示应明确说明增强搜索失败并已回退
- [ ] 页面不应出现空白、随机演示数据或旧 `UP云搜` 入口

## 4. 缓存隔离验收

### 4.1 新命名空间
- [ ] 打开浏览器开发者工具
- [ ] 查看当前站点 `sessionStorage`
- [ ] 确认存在 `796h-mc-schema-version = 2.0.8`
- [ ] 确认新的缓存 key 前缀为 `796h-mc-2.0.8-`

### 4.2 按增强服务签名隔离
- [ ] 先在无增强服务状态下搜索一次
- [ ] 再填入一个增强服务地址并重新搜索同一关键词

- [ ] 清空增强搜索配置后再次搜索
- [ ] 确认页面重新回到默认站外搜索结果，而不是继续命中增强结果缓存
