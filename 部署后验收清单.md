# 796Helper 影视搜索部署与验收清单

> 目标：确认 `v2.0.6` 已真正生效，影视搜索链路能够通过 Worker 直接抓取 `PanSearch` 并返回真实网盘链接；当前端代理不可用时，仅回退到 `PanSearch` 搜索页；旧缓存不再污染结果。

## 0. 发布前确认
- [ ] 本地代码已包含 `v2.0.6` 调整
- [ ] `worker/index.js` 已优先解析 `PanSearch` 的 `__NEXT_DATA__`
- [ ] `worker/index.js` 在 `__NEXT_DATA__` 未命中时仍保留 HTML 兜底提取
- [ ] `js/pages/movie-search.js` 已保持“真实直链优先、失败时仅回退 `PanSearch` 搜索页”的行为
- [ ] `index.html` 中 `window.__796HELPER_CONFIG__.movieSearch.apiBases` 已确认目标 Worker 地址
- [ ] 已准备 Cloudflare Workers 发布权限
- [ ] 已准备 GitHub 仓库推送权限
- [ ] 已准备 3 个测试关键词：
  - [ ] `哪吒2`
  - [ ] `流浪地球2`
  - [ ] `三体`

## 1. Worker 部署步骤

### 1.1 发布 Worker
1. 打开终端，进入项目 Worker 目录：`cd worker`
2. 若本机未安装 Wrangler，先执行：`npm install -g wrangler`
3. 首次发布或登录失效时执行：`wrangler login`
4. 发布最新 Worker：`wrangler deploy`
5. 记录控制台返回的 Worker 地址，当前默认可为：`https://796helper-movie-search.clown8379.workers.dev`

### 1.2 部署后立即冒烟检查
- [ ] 打开：`https://796helper-movie-search.clown8379.workers.dev/health`
- [ ] 确认返回 HTTP 200
- [ ] 确认返回 JSON
- [ ] 确认 `status = "ok"`
- [ ] 确认 `version = "2.0.6"`
- [ ] 确认 `sources = ["pansearch"]`
- [ ] 确认 `routes` 中包含 `/api/search` 与 `/health`

### 1.2.1 若 `/health` 无法打开，先做访问层排查
- [ ] 执行：`wrangler whoami`
- [ ] 确认当前部署账号正确（本项目当前应为 `clown8379@163.com`）
- [ ] 执行：`wrangler deployments status --name 796helper-movie-search --json`
- [ ] 确认生产流量已经指向最新 `Version ID`
- [ ] 执行：`Resolve-DnsName 796helper-movie-search.clown8379.workers.dev`
- [ ] 执行：`Resolve-DnsName workers.dev`
- [ ] 若返回明显非 Cloudflare 的 IP，或 HTTPS 请求直接报 TLS/连接关闭错误，优先判断为本地网络对 `workers.dev` 的 DNS/TLS 干扰
- [ ] 该场景下不要先改 `worker/index.js`；优先换网络、开启安全 DNS/DoH，或绑定 Cloudflare 自定义域名作为正式入口
- [ ] 若已绑定自定义域名，将该域名放到 `index.html` 与 `js/pages/movie-search.js` 的 `apiBases` 第一位

### 1.3 若 Worker 地址变更，再改前端运行时配置

- [ ] 打开 `index.html`
- [ ] 找到 `window.__796HELPER_CONFIG__.movieSearch.apiBases`
- [ ] 将最新地址放在数组第一位
- [ ] 若保留多代理，确保优先级从前到后排列
- [ ] 保持 `enableExternalFallback: true`

## 4. API 结果验收

### 4.1 真实网盘直链
- [ ] 打开：`https://796helper-movie-search.clown8379.workers.dev/api/search?keyword=哪吒2`
- [ ] 检查 `success = true`
- [ ] 检查 `version = "2.0.6"`
- [ ] 检查 `data` 为数组
- [ ] 若有结果，前几条应优先出现真实网盘域名
- [ ] 允许出现的直链域名示例：
  - [ ] `pan.baidu.com`
  - [ ] `drive.quark.cn`
  - [ ] `alipan.com` / `aliyundrive.com`
  - [ ] `pan.xunlei.com`
  - [ ] `cloud.189.cn`
  - [ ] `115.com`
  - [ ] `lanzou*`

### 4.2 `PanSearch` 抓取链路
- [ ] 确认当前版本以 `PanSearch` 为唯一上游
- [ ] 确认 Worker 优先解析 `__NEXT_DATA__`
- [ ] 确认真实网盘链接来自 `props.pageProps.data.data[].content`
- [ ] 若 `__NEXT_DATA__` 缺失或结构变化，仍可回退到 HTML 文本/属性提取

### 4.3 字段完整性
- [ ] 每条结果至少包含：`title`、`source`、`sourceLabel`、`link`
- [ ] 若链接带 `pwd`，则 `code` 应同步返回
- [ ] `total` 与 `data.length` 一致或大于 0

### 4.4 PanSearch-only 约束
- [ ] API 返回结果中不应再出现 `upyunso.com`
- [ ] API 返回结果中不应再出现 `pansearch.me/search?...` 这类搜索页链接
- [ ] 若关键词无可用直链，允许 `data = []`，但不应伪造 `UP云搜` 兜底结果

失败判定：
- 返回 `success = false`
- 明明存在直链资源，但 API 结果仍全部为搜索页链接
- 结果中仍出现 `upyunso.com`
- 结果中出现前端兜底用的 `PanSearch` 搜索页链接

## 5. 前端页面验收

### 5.1 首次打开与搜索
- [ ] 打开前端页面并进入 `影视搜索`
- [ ] 搜索 `哪吒2`
- [ ] 页面能正常展示结果卡片
- [ ] 结果卡片按钮文案与链接类型一致
- [ ] 顶部未出现与 `UP云搜` 相关的提示或入口

### 5.2 打开链接行为
- [ ] 点击真实网盘结果的“打开网盘链接”按钮
- [ ] 新标签页应直接打开对应网盘站点
- [ ] 至少验证 2 条不同来源的结果
- [ ] 不应跳转到 `UP云搜` 搜索页

### 5.3 复制行为
- [ ] 对真实网盘结果点击“复制网盘链接”
- [ ] 剪贴板内容应为：直链 + 提取码（若存在）
- [ ] 若页面进入前端兜底状态，再点击复制
- [ ] 剪贴板内容应为：资源标题 + `PanSearch` 搜索页链接

### 5.4 来源筛选
- [ ] 切换 `全部 / 百度 / 夸克 / 阿里 / 迅雷 / 天翼` 等来源筛选
- [ ] 每次切换后结果能重新请求并正常展示
- [ ] 筛选后结果来源与所选类型基本一致

## 6. 前端兜底验收

### 6.1 触发方式
可任选一种方式触发：
- 暂时把 `index.html` 中 `apiBases` 改成无效地址后本地打开页面
- 或临时让 Worker 返回不可用状态
- 或在浏览器开发者工具中模拟请求失败

### 6.2 预期行为
- [ ] 页面不崩溃
- [ ] 结果区出现明确提示：已切换为 `PanSearch` 搜索页
- [ ] 结果区只出现 1 张 `PanSearch` 搜索页入口卡片
- [ ] 不应再出现 `UP云搜` 搜索页卡片
- [ ] 点击“打开”应跳到 `PanSearch` 搜索结果页

失败判定：
- 前端仍生成 `UP云搜` 搜索页卡片
- 失败提示仍提到 `UP云搜`
- 页面回退到随机演示数据或空白状态

## 7. 缓存隔离验收

### 7.1 新缓存命名空间
- [ ] 打开浏览器开发者工具
- [ ] 查看当前站点 `sessionStorage`
- [ ] 确认存在 `796h-mc-schema-version = 2.0.6`
- [ ] 确认新的缓存 key 前缀为 `796h-mc-2.0.6-`
- [ ] 确认旧前缀缓存未继续命中或会被自动清理

### 7.2 重复搜索
- [ ] 同一关键词连续搜索两次
- [ ] 第二次结果仍保持真实网盘直链
- [ ] 刷新页面后再次搜索，不应回退成旧 `UP云搜` 搜索页结果

失败判定：
- `sessionStorage` 中仍主要命中旧 `796h-mc-` 结果
- 刷新后结果退化为旧搜索页链接

## 8. 异常与回归验收

### 8.1 空结果 / 超时
- [ ] 输入一个极冷门关键词
- [ ] 页面不崩溃
- [ ] 若接口超时，应显示可理解的失败提示
- [ ] 失败时只允许回退到 `PanSearch` 搜索页

### 8.2 路由与交互回归
- [ ] 普通搜索功能可用
- [ ] 路由切换后再次进入 `影视搜索`，按钮点击仍有效
- [ ] 复制按钮、筛选按钮、清空按钮均正常工作
- [ ] 其他页面功能未受影响

## 9. 通过标准
满足以下条件即可判定部署验收通过：
- [ ] `/health` 返回 `version = 2.0.6`
- [ ] `/health` 返回 `sources = ["pansearch"]`
- [ ] API 结果已不再包含 `UP云搜` 或搜索页链接
- [ ] API 能通过 `PanSearch` 抓取逻辑返回真实网盘直链
- [ ] 前端点击直链时，实际打开的是网盘站点
- [ ] 代理失败时，前端只回退到 `PanSearch` 搜索页
- [ ] `sessionStorage` 已切换到 `796h-mc-2.0.6-` 命名空间
- [ ] 无明显回归问题

## 10. 若验收失败，优先排查
1. Worker 是否未部署到最新版本
2. `worker/index.js` 是否仍停留在 `v2.0.5` 之前的旧逻辑
3. `index.html` 是否仍指向错误或过期的 Worker 地址
4. GitHub Pages 是否仍在提供旧静态资源
5. 浏览器是否仍保留旧 `sessionStorage`
6. `PanSearch` 上游页面结构是否再次变更
7. 当前测试关键词是否刚好无可用直链
8. 是否误把前端兜底搜索页结果当成 API 真实返回
